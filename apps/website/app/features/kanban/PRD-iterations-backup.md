# PRD: 채용 칸반 보드

## Iteration 1

### 목표

채용 전형별 칸반 보드 페이지를 구현한다. 하나의 채용 공고에 여러 전형(서류심사, 1차 면접, 2차 면접, 최종합격)이 존재하고, 지원자 카드를 드래그하여 전형 간 이동하거나 컨텍스트 메뉴로 액션을 수행할 수 있다. 이 이터레이션에서는 더미 데이터를 사용한다.

### 기능 명세

1. **칸반 보드 표시**
   - 채용 공고 정보(제목, 부서, 상태)를 헤더에 표시
   - 전형별 컬럼을 가로로 나열 (서류심사 → 1차 면접 → 2차 면접 → 최종합격)
   - 각 컬럼 헤더에 전형 이름과 해당 전형의 지원자 수를 표시

2. **지원자 카드**
   - 아바타, 이름, 태그(경력/신입 등) 표시
   - 우클릭 시 컨텍스트 메뉴 표시
   - 카드 내 더보기 버튼 클릭 시 드롭다운 메뉴 표시
   - 메뉴 항목: 메시지 보내기, 메모 추가, 이력서 보기, 탈락 처리

3. **드래그 앤 드롭**
   - 지원자 카드를 드래그하여 같은 컬럼 내 순서 변경
   - 지원자 카드를 드래그하여 다른 컬럼으로 이동
   - 드롭 위치에 인디케이터 표시
   - 빈 컬럼에도 드롭 가능

### UI/UX 명세

- **레이아웃**: 전체 화면 너비 사용, 가로 스크롤 가능
- **보드 헤더**: 채용 제목(h1), 부서명, 상태 Badge
- **컬럼**: 고정 너비(320px), 세로 스크롤 가능한 카드 리스트
- **카드**: Card 컴포넌트 사용, hover 시 약간의 그림자 변화
- **드래그 피드백**: 드래그 중 카드 반투명(opacity-50), 드롭 타겟 컬럼 배경색 변경, 드롭 인디케이터(파란색 라인)
- **컨텍스트 메뉴**: 우클릭으로 접근, DropdownMenu와 동일한 메뉴 항목

### 마일스톤 기준

- [x] `/recruitment` 경로에서 칸반 보드 페이지가 렌더링된다
- [x] 4개 전형 컬럼이 표시된다 (서류심사, 1차 면접, 2차 면접, 최종합격)
- [x] 각 컬럼에 지원자 카드가 표시된다
- [x] 카드를 드래그하여 같은 컬럼 내에서 순서를 변경할 수 있다
- [x] 카드를 드래그하여 다른 컬럼으로 이동할 수 있다
- [x] 빈 컬럼에 카드를 드롭할 수 있다
- [x] 드래그 중 시각적 피드백이 표시된다 (반투명, 드롭 인디케이터)
- [x] 카드 우클릭 시 컨텍스트 메뉴가 표시된다
- [x] 카드 더보기 버튼 클릭 시 드롭다운 메뉴가 표시된다
- [x] `vp check` 통과

### 제약 사항

- 백엔드 API 없이 클라이언트 사이드 더미 데이터만 사용
- 메뉴 액션(메시지 보내기 등)은 console.log로 처리
- 전형 컬럼 순서는 고정 (드래그로 컬럼 이동 불가)
- 반응형 디자인은 스코프 밖 (데스크톱 해상도 기준)

## Iteration 2

### 목표

칸반 보드의 모든 인터랙션(드래그, 드롭, 메뉴 액션)을 시각적으로 확인할 수 있는 개발자용 로그 패널을 추가한다. 기존에 `console.log`로만 출력되던 인터랙션 이벤트를 화면 하단 서랍 패널에 실시간으로 표시한다.

### 기능 명세

1. **이벤트 로그 시스템**
   - React Context + useReducer로 로그 상태 관리
   - 로그 이벤트 타입:
     - `DRAG_START` — 드래그 시작 (candidateId, sourceStageId)
     - `DRAG_END` — 드래그 종료 (candidateId)
     - `CARD_REORDER` — 같은 컬럼 내 순서 변경 (candidateId, stageId, fromIndex, toIndex)
     - `CARD_MOVE` — 다른 컬럼으로 이동 (candidateId, sourceStageId, destStageId)
     - `DROP_ON_EMPTY` — 빈 컬럼에 드롭 (candidateId, sourceStageId, destStageId)
     - `MENU_ACTION` — 메뉴 액션 실행 (action, candidateId)
   - 각 로그 엔트리: `{ id, type, timestamp, data, message }`

2. **하단 서랍 패널**
   - 화면 하단에 고정된 서랍 패널
   - 토글 버튼으로 열기/닫기 (기본 닫힘)
   - 로그 목록을 시간 역순으로 표시 (최신이 위)
   - 각 로그 엔트리에 타임스탬프, 이벤트 타입 뱃지, 설명 메시지 표시
   - 로그 초기화(Clear) 버튼
   - ScrollArea로 스크롤 가능

3. **기존 코드 통합**
   - `use-kanban-board.ts`의 드래그/드롭 이벤트에 로그 호출 삽입
   - `kanban-card.tsx`의 드래그 시작/종료에 로그 호출 삽입
   - `candidate-actions.tsx`의 메뉴 액션에 로그 호출 삽입

### UI/UX 명세

- **패널 위치**: 화면 하단 고정, 칸반 보드 위에 오버레이
- **토글 버튼**: 패널 상단에 위치, 로그 건수 표시
- **패널 높이**: 열린 상태에서 300px
- **로그 엔트리**: 타임스탬프(HH:MM:SS), 이벤트 타입 Badge, 메시지 텍스트
- **이벤트 타입 뱃지 색상**: 드래그 계열(blue), 이동 계열(green), 메뉴 계열(purple)

### 마일스톤 기준

- [x] 카드 드래그 시작/종료 시 패널에 로그가 표시된다
- [x] 같은 컬럼 내 순서 변경 시 CARD_REORDER 로그가 표시된다
- [x] 다른 컬럼으로 이동 시 CARD_MOVE 로그가 표시된다
- [x] 빈 컬럼에 드롭 시 DROP_ON_EMPTY 로그가 표시된다
- [x] 메뉴 액션 클릭 시 MENU_ACTION 로그가 표시된다
- [x] Clear 버튼으로 로그를 초기화할 수 있다
- [x] 패널 토글 버튼으로 열기/닫기가 동작한다
- [x] `vp check` 통과

### 제약 사항

- 로그는 클라이언트 메모리에만 저장 (새로고침 시 초기화)
- 개발용 패널이므로 프로덕션 빌드에서 제거하는 것은 이번 스코프 밖
- 로그 최대 개수 제한 없음 (메모리 관리는 이후 이터레이션에서 고려)

## Iteration 3

### 목표

체크박스로 지원자를 다중 선택하고, 선택 시 하단 벌크 액션 툴바를 표시하며, 선택된 카드들을 한번에 드래그앤드롭으로 이동하는 기능을 추가한다. 기존 React Context 기반 로그 시스템을 zustand 스토어로 마이그레이션하여 상태 관리 방식을 통일한다.

### 기능 명세

1. **다중 선택**
   - 각 지원자 카드에 체크박스 표시 (항상 노출)
   - 체크박스 클릭으로 선택/해제 토글
   - 선택된 카드에 시각적 강조 (ring-2 ring-primary/50)
   - zustand 스토어로 선택 상태 관리

2. **벌크 액션 툴바**
   - 1명 이상 선택 시 화면 하단에 툴바 표시
   - 표시 내용: "{N}명 선택됨", 메시지 보내기, 메모 추가, 이력서 보기, 탈락 처리, 전체 해제
   - 액션 클릭 시 Dev Log에 MENU_ACTION (bulk) 로그 기록

3. **벌크 드래그앤드롭**
   - 선택된 카드를 드래그하면 선택된 카드 전체가 이동
   - 벌크 드래그 중 나머지 선택 카드는 opacity-50 표시
   - 벌크 이동 완료 후 선택 상태 자동 초기화
   - 비선택 카드를 드래그하면 기존과 동일하게 단건 이동

4. **로그 시스템 zustand 마이그레이션**
   - React Context + useReducer → zustand 스토어 교체
   - `BULK_MOVE`, `BULK_DROP_ON_EMPTY` 이벤트 타입 추가
   - 기존 `useKanbanLog()` 훅 인터페이스 유지

### UI/UX 명세

- **체크박스 위치**: CardContent 좌측, 이름 왼쪽
- **선택 시각 효과**: `ring-2 ring-primary/50` 테두리
- **벌크 드래그 중**: 선택된 다른 카드 `opacity-50`
- **벌크 액션 툴바**: 화면 하단 고정, DevPanel 위에 배치
- **하단 영역**: 단일 fixed 컨테이너 — BulkActionToolbar (조건부) + DevPanel
- **벌크 이벤트 뱃지 색상**: orange (기존: blue=드래그, green=이동, purple=메뉴)

### 마일스톤 기준

- [x] 체크박스 클릭으로 카드 선택/해제가 동작한다
- [x] 선택된 카드에 시각적 강조가 표시된다
- [x] 1명 이상 선택 시 하단 벌크 액션 툴바가 표시된다
- [x] 툴바 액션 클릭 시 Dev Log에 MENU_ACTION (bulk) 로그가 표시된다
- [x] 선택된 카드를 드래그하면 선택된 카드 전체가 이동한다
- [x] 벌크 이동 후 선택 상태가 초기화된다
- [x] 전체 해제 버튼이 동작한다
- [x] 비선택 카드 드래그 시 단건 이동이 유지된다
- [x] 로그 시스템이 zustand 스토어로 동작한다
- [x] `vp check` 통과

### 제약 사항

- 벌크 액션은 로그 기록만 수행 (실제 메시지 전송 등은 미구현)
- 키보드 단축키(Shift+클릭 범위 선택 등)는 스코프 밖
- 벌크 드래그 시 드래그 프리뷰는 기본 프리뷰 사용 (커스텀 프리뷰는 이후 이터레이션에서 고려)

## Iteration 4

### 목표

12개 전형 × 1,000명 규모의 대규모 데이터를 가상화로 처리하고, API 도입을 대비한 optimistic update 패턴을 구축한다.

### 기능 명세

1. **대규모 더미 데이터 생성**
   - 12개 전형: 서류심사, 코딩테스트, 1차 면접, 과제전형, 2차 면접, 임원면접, 레퍼런스체크, 처우협상, 건강검진, 최종합격, 입사대기, 입사완료
   - 각 전형 1,000명 (총 12,000명)
   - 한국 이름 풀(성 20 × 이름 20) + 스킬 태그를 deterministic하게 조합

2. **Optimistic Update 인프라**
   - `useBoardMutation` 훅: snapshot → optimistic apply → (optional) API → rollback on failure
   - 순수 함수로 상태 변환 로직 분리
   - `apiFn` 없으면 동기 동작 (기존 동작과 동일)
   - API 통합 시 `apiFn`만 추가하면 optimistic update 완성

3. **컬럼 가상화**
   - `@tanstack/react-virtual`의 `useVirtualizer`로 컬럼 내 카드 가상화
   - Radix ScrollArea → plain div + `overflow-y: auto`
   - `autoScrollForElements`로 드래그 중 세로 자동 스크롤
   - absolute positioning으로 가상 아이템 렌더링

4. **보드 레이아웃 최적화**
   - `monitorForElements` deps에서 `boardState` 제거 → `boardStateRef` 사용
   - 보드 가로 스크롤 영역에 `autoScrollForElements` 등록

### UI/UX 명세

- 12개 컬럼이 가로 스크롤로 표시됨
- 각 컬럼 내 1,000개 카드가 가상화되어 부드러운 스크롤
- 드래그 중 세로(컬럼 내) + 가로(보드) 자동 스크롤
- 기존 체크박스 선택, 벌크 드래그, 로그 패널 모두 유지

### 마일스톤 기준

- [x] 12개 컬럼 × 1,000명 렌더링 확인
- [x] 스크롤 시 부드러운 가상화 동작
- [x] 가상화 리스트에서 단건 드래그앤드롭 동작
- [x] 가상화 리스트에서 벌크 드래그앤드롭 동작
- [x] 드래그 중 세로 자동 스크롤 동작
- [x] 드래그 중 가로 자동 스크롤 동작
- [x] 체크박스 선택/벌크 툴바 기존 기능 유지
- [x] Dev Log에 이벤트 정상 기록
- [x] `vp check` 통과

### 제약 사항

- 카드 높이는 고정 estimateSize(72px) 사용 (가변 높이는 필요 시 추후 도입)
- Radix ScrollArea 제거 후 CSS `scrollbar-*` 속성으로 스크롤바 스타일 보완
- 화면 밖 선택 카드는 DOM에 없지만 zustand store에 상태 유지

## Iteration 5

### 목표

메인 칸반보드의 각 전형 내부에 세부 액션 단계를 도입하고, 전형 헤더 클릭 시 액션별 칸반보드로 진입할 수 있도록 구현한다.

### 기능 명세

1. **전형별 액션 데이터**
   - 12개 전형에 각각 3~5개의 세부 액션 단계를 정의
     - 서류심사: 접수완료 → 서류검토중 → 서류합격 → 결과통보
     - 코딩테스트: 시험발송 → 시험진행중 → 채점중 → 채점완료
     - 면접류: 일정조율 → 면접대기 → 면접완료 → 평가중
     - 기타 전형도 동일한 패턴으로 3~4개 액션 정의
   - 각 지원자에게 `currentActionId`를 부여하여 현재 액션 단계 표시

2. **메인 보드 카드 액션 표시**
   - 기존 카드 디자인(이름 + 태그 + 체크박스 + 메뉴) 유지
   - 태그 아래에 현재 액션 정보를 한 줄 추가 (`→ 서류검토중` 형태, text-[11px], muted 컬러)

3. **컬럼 헤더 링크**
   - 메인 보드의 전형 헤더(전형명)를 클릭하면 `/recruitment/:stageId`로 이동
   - 호버 시 밑줄 표시

4. **전형 상세 칸반보드**
   - 상단에 뒤로가기 버튼 + 전형명 + 지원자 수 표시
   - 액션별 컬럼으로 구성된 칸반보드 렌더링
   - 기존 `KanbanColumn`, `KanbanCard`, `BulkActionToolbar`, `DevPanel` 재사용
   - 드래그앤드롭 (단건 + 벌크) 지원
   - 가상화 스크롤 적용

## Iteration 6

### 목표

컬럼 헤더에 전체선택 체크박스를 추가하고, 카드를 다른 컬럼으로 이동할 때 액션 정보가 유실되는 버그를 수정한다.

### 기능 명세

1. **컬럼 전체선택 체크박스**
   - selection store에 `toggleAll(ids)` 메서드 추가
   - 전달된 ids가 모두 selectedIds에 포함되어 있으면 전부 제거, 하나라도 빠져 있으면 전부 추가
   - 컬럼 헤더 왼쪽에 `Checkbox` 컴포넌트 배치
   - 체크 상태: 모두 선택 → checked, 일부 선택 → indeterminate, 없음 → unchecked
   - 클릭 시 `toggleAll(candidateIds)` 호출

2. **메인 보드 액션 유실 버그 수정**
   - `moveCardPure`: 크로스 스테이지 이동 시 `candidates[id].currentActionId`를 대상 스테이지의 첫 번째 액션 ID로 업데이트
   - `moveCardsPure`: 벌크 이동 대상 전원의 `currentActionId`를 대상 스테이지 첫 액션으로 업데이트
   - 같은 스테이지 내 reorder는 변경 없음

3. **상세 보드 액션 유실 버그 수정**
   - `moveCardPure`: 다른 액션으로 이동 시 `currentActionId = destinationActionId`
   - `moveCardsPure`: 벌크 이동 대상 전원의 `currentActionId = destActionId`

### UI/UX 명세

- 컬럼 헤더 레이아웃: `[✓] 전형명 [카운트]`
- 체크박스는 해당 컬럼에 카드가 있을 때만 표시
- 카드 이동 후 새 스테이지/액션에 맞는 액션 태그가 카드에 표시됨
