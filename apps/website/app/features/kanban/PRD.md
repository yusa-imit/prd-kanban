# PRD: 채용 칸반 보드

## 제품 개요

채용 전형별 칸반 보드 애플리케이션. 하나의 채용 공고에 12개 전형이 존재하고, 지원자 카드를 드래그하여 전형 간 이동하거나 컨텍스트 메뉴로 액션을 수행할 수 있다. 각 전형은 세부 액션 단계를 가지며, 전형 헤더 클릭으로 액션별 상세 칸반보드에 진입할 수 있다.

## 기능 목록

### 칸반 보드 & 12개 전형 컬럼

- 채용 공고 정보(제목, 부서, 상태)를 헤더에 표시
- 12개 전형 컬럼을 가로로 나열: 서류심사, 코딩테스트, 1차 면접, 과제전형, 2차 면접, 임원면접, 레퍼런스체크, 처우협상, 건강검진, 최종합격, 입사대기, 입사완료
- 각 컬럼 헤더에 전형 이름과 해당 전형의 지원자 수 표시
- 컬럼 헤더 클릭 시 `/recruitment/:stageId` 상세 보드로 이동 (호버 시 밑줄)

### 지원자 카드

- 아바타, 이름, 태그(경력/신입 등) 표시
- 태그 아래에 현재 액션 정보 한 줄 표시 (`→ 서류검토중` 형태, text-[11px], muted 컬러)
- 우클릭 시 컨텍스트 메뉴 표시
- 카드 내 더보기 버튼 클릭 시 드롭다운 메뉴 표시
- 메뉴 항목: 메시지 보내기, 메모 추가, 이력서 보기, 탈락 처리

### 드래그 앤 드롭 (단건/벌크)

- 카드를 드래그하여 같은 컬럼 내 순서 변경, 다른 컬럼으로 이동, 빈 컬럼에 드롭 가능
- 드롭 위치에 인디케이터 표시
- 드래그 중 시각적 피드백 (반투명 opacity-50, 드롭 타겟 배경색 변경)
- 선택된 카드를 드래그하면 선택된 카드 전체가 이동 (벌크 드래그)
- 벌크 드래그 중 나머지 선택 카드는 opacity-50 표시
- 벌크 이동 완료 후 선택 상태 자동 초기화
- 비선택 카드 드래그 시 단건 이동 유지
- 카드 이동 시 `currentActionId`를 대상 스테이지/액션의 첫 번째 액션 ID로 자동 업데이트

### 컨텍스트 메뉴 & 드롭다운 메뉴

- 우클릭으로 컨텍스트 메뉴, 더보기 버튼으로 드롭다운 메뉴
- 동일한 메뉴 항목 (메시지 보내기, 메모 추가, 이력서 보기, 탈락 처리)
- 액션 클릭 시 Dev Log에 MENU_ACTION 로그 기록

### Dev Log 패널

- 화면 하단 고정 서랍 패널 (기본 닫힘, 토글 버튼으로 열기/닫기)
- 로그 건수 표시, Clear 버튼, ScrollArea로 스크롤 가능
- 이벤트 타입: DRAG_START, DRAG_END, CARD_REORDER, CARD_MOVE, DROP_ON_EMPTY, MENU_ACTION, BULK_MOVE, BULK_DROP_ON_EMPTY
- 이벤트 타입 뱃지 색상: 드래그(blue), 이동(green), 메뉴(purple), 벌크(orange)
- zustand 스토어로 상태 관리

### 체크박스 선택 & 벌크 액션 툴바

- 각 카드에 체크박스 표시 (항상 노출)
- 체크박스 클릭으로 선택/해제 토글, 선택된 카드에 ring-2 ring-primary/50 강조
- 1명 이상 선택 시 화면 하단 벌크 액션 툴바 표시
- 표시 내용: "{N}명 선택됨", 메시지 보내기, 메모 추가, 이력서 보기, 탈락 처리, 전체 해제
- zustand 스토어로 선택 상태 관리

### 컬럼 전체선택 (indeterminate 포함)

- 컬럼 헤더 왼쪽에 Checkbox 컴포넌트 배치 (카드가 있을 때만 표시)
- 체크 상태: 모두 선택 → checked, 일부 선택 → indeterminate, 없음 → unchecked
- 클릭 시 `toggleAll(candidateIds)` 호출 (가상화된 카드 포함)

### 가상화 (12,000명)

- 12개 전형 × 1,000명 (총 12,000명) 규모 처리
- `@tanstack/react-virtual`의 `useVirtualizer`로 컬럼 내 카드 가상화
- 고정 estimateSize(72px), absolute positioning으로 가상 아이템 렌더링
- `autoScrollForElements`로 드래그 중 세로(컬럼 내) + 가로(보드) 자동 스크롤

### 전형 상세 보드 (액션별 칸반)

- 전형별 3~5개 세부 액션 단계 (서류심사: 접수완료→서류검토중→서류합격→결과통보, 등)
- 상단에 뒤로가기 버튼 + 전형명 + 지원자 수 표시
- 액션별 컬럼으로 구성된 칸반보드 렌더링
- 기존 KanbanColumn, KanbanCard, BulkActionToolbar, DevPanel 재사용
- 드래그앤드롭 (단건 + 벌크) 지원, 가상화 스크롤 적용
- 카드 이동 시 `currentActionId`를 대상 액션 ID로 업데이트

### Optimistic Update 인프라

- `useBoardMutation` 훅: snapshot → optimistic apply → (optional) API → rollback on failure
- 순수 함수로 상태 변환 로직 분리
- `apiFn` 없으면 동기 동작, API 통합 시 `apiFn`만 추가하면 완성

## 기술 스택

- **더미 데이터**: 한국 이름 풀(성 20 × 이름 20) + 스킬 태그를 deterministic하게 조합
- **상태 관리**: zustand (보드 상태, 선택 상태, 로그 상태)
- **가상화**: @tanstack/react-virtual
- **드래그앤드롭**: @atlaskit/pragmatic-drag-and-drop
- **라우팅**: React Router 7 (framework mode)
- **UI**: shadcn/ui 기반 디자인 시스템 (`packages/ui`)

## 제약 사항

- 백엔드 API 없이 클라이언트 사이드 더미 데이터만 사용
- 메뉴 액션(메시지 보내기 등)은 Dev Log 기록만 수행 (실제 동작 미구현)
- 전형 컬럼 순서는 고정 (드래그로 컬럼 이동 불가)
- 반응형 디자인은 스코프 밖 (데스크톱 해상도 기준)
- 키보드 단축키(Shift+클릭 범위 선택 등)는 스코프 밖
- 카드 높이는 고정 estimateSize(72px) 사용
- 로그는 클라이언트 메모리에만 저장 (새로고침 시 초기화)
